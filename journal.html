Historique des problèmes

Erreur de Configuration EFI

Lors de la première tentative, la VM ne démarrait pas. EndeavourOS (basé sur Arch) nécessite une table de partitionnement GPT moderne, qui est incompatible avec le mode BIOS (Legacy) par défaut de VirtualBox.

Solution : Cocher "Enable EFI" dans les paramètres système de la VM.

Instabilité Graphique (Black Screen)

Après l'installation, l'écran de connexion (SDDM) restait noir ou figeait. Ce conflit est dû à la gestion de l'accélération 3D avec le GPU hybride de l'hôte Asus G14.

Solution : Passage au contrôleur VMSVGA, allocation de 128 Mo de VRAM et désactivation de l'accélération 3D.

Erreur de Redirection de Port (NAT)

Nous avons initialement tenté d'utiliser le port 8883 pour accéder au serveur web depuis l'hôte, mais cela posait des problèmes de standardisation pour les tests.

Solution : Correction de la règle NAT pour utiliser le port standard 8080 vers le port 80 de la VM.
