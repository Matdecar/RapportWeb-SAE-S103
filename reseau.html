I. Service Web (Apache)

Le serveur HTTP Apache est la pierre angulaire de notre infrastructure. Nous l'avons installé via le gestionnaire de paquets officiel d'Arch Linux en utilisant la commande sudo pacman -S apache. Cette étape déploie les binaires et les fichiers de configuration par défaut.

Une fois installé, le service ne démarre pas automatiquement. Nous avons utilisé systemctl enable --now httpd pour non seulement démarrer le serveur immédiatement, mais aussi garantir qu'il se relance automatiquement à chaque redémarrage de la VM.

II. Stratégie de Mise en Ligne

Phase 1 : Test en NAT (Port Forwarding)

Avant d'exposer le serveur sur le réseau public, nous avons validé son fonctionnement en local. Nous avons mappé le port 8080 de l'hôte (Asus G14) vers le port 80 de la VM. Cela permet d'accéder au site via http://127.0.0.1:8080 sans modifier la configuration réseau de l'établissement.

Phase 2 : Production (Mode Bridged)

Pour répondre à l'exigence de rendre le site "accessible par toute personne sur le même réseau", nous avons basculé l'interface réseau en mode Accès par Pont (Bridged Adapter) sur la carte Wi-Fi. La VM obtient ainsi sa propre adresse IP sur le réseau local, la rendant visible comme une machine physique distincte.

III. Workflow de Développement

Pour optimiser le flux de travail, nous avons monté un dossier partagé. Cela permet de développer le code HTML/CSS sur l'hôte avec l'IDE PhpStorm (profitant des performances du Ryzen 9) tout en servant les fichiers instantanément depuis le répertoire /srv/http/ de la VM.
