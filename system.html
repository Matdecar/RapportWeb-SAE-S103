I. Préparation de l'Hôte

Avant de virtualiser, nous avons préparé l'hôte Asus G14. L'installation des paquets virtualbox et virtualbox-host-modules-arch via pacman est cruciale pour assurer la communication directe entre l'hyperviseur et le noyau Linux-g14 spécifique à la machine.

Gestion des droits : L'ajout de l'utilisateur au groupe vboxusers est une étape de sécurisation qui permet de piloter les interfaces réseaux (pour le mode Pont) et les périphériques USB sans élever les privilèges en permanence via sudo.

sudo usermod -aG vboxusers $USER

II. Architecture et Dimensionnement

Ressources "Confort"

Grâce aux 32 Go de RAM de l'hôte, nous avons configuré une machine virtuelle robuste :

  • 8 Go de RAM : Garantit une fluidité parfaite de l'interface Plasma.
  • 4 vCPUs : Assure le traitement parallèle des requêtes Apache.
Le choix de l'EFI

L'activation de l'option "Use EFI" est techniquement indispensable pour les distributions Arch modernes (EndeavourOS). Sans cela, le chargeur d'amorçage ne peut pas gérer correctement la table de partition GPT du disque virtuel de 50 Go.
